## Why

Agentic coding — Claude Code, opencode, Aider, and the dev servers
and test runners they drive — lives in the terminal. But the terminal
is stuck where you started it: one laptop, one window size, one SSH
hop. Walk away, switch devices, or lose the connection and you're
rebuilding context from scratch.

portagenty treats your agent sessions as a **portable workspace**.
Define them once in a committable TOML file, launch them on any
device over tmux or zellij, and hop between laptop, desktop, and
phone-over-SSH (Termux) with a single `pa claim`. The sessions keep
running; only the terminal attached to them changes.

## Features

<CardGrid>
  <Card title="One-command install" icon="rocket">
    `cargo install --git https://github.com/cybersader/portagenty`.
    No clone, no `cd`, no hand compile. Single static Rust binary
    at `~/.cargo/bin/pa`.
  </Card>
  <Card title="Cross-device takeover" icon="seti:config">
    SSH in from your phone, run `pa claim`, and the session instantly
    reshapes to the new device. No more "stuck at my laptop's
    terminal width" after device hops.
  </Card>
  <Card title="Untracked session adoption" icon="approve-check">
    Tmux / zellij sessions you started manually show up in the TUI
    alongside workspace-defined ones. Attach with Enter — nothing
    new to set up.
  </Card>
  <Card title="Multiplexer-aware" icon="setting">
    tmux + zellij first-class. Sessions with `cwd`, `command`, and
    `env` work consistently across both. WezTerm explicitly skipped
    — see the roadmap.
  </Card>
  <Card title="First-run wizard" icon="open-book">
    Run `pa` on a new machine with no config and it walks you
    through a 30-second setup. No TOML editing required to start.
  </Card>
  <Card title="No-editor session management" icon="pencil">
    `pa init` / `pa add` / `pa rm` / `pa edit` handle the whole
    workspace-file lifecycle from the command line. Phone-keyboard
    friendly, comment-preserving.
  </Card>
  <Card title="Bundled bash ergonomics" icon="puzzle">
    `pa snippets install pa-aliases` drops curated, idempotent
    shell snippets into your rc file. Short aliases, Termux-friendly
    tweaks, all opt-in.
  </Card>
  <Card title="Committable layouts" icon="document">
    `pa export` renders your workspace as a tmux starter script or
    zellij KDL layout. Commit it; teammates launch without needing
    `pa` installed.
  </Card>
  <Card title="Find folder + scaffold" icon="magnifier">
    Press `n` in the workspace picker to fuzzy-search your filesystem
    for project folders. Tiered backends (recency, zoxide, plocate,
    fd, stdlib walker) ranked by nucleo. Pick a folder and `pa`
    scaffolds a workspace TOML on the spot.
  </Card>
  <Card title="Tree browser" icon="seti:folder">
    Press `t` inside the find overlay to switch to a filesystem tree
    view. Expand/collapse directories, navigate with arrow keys or
    vim bindings, select a folder with Shift+Enter. Used for both new
    workspace creation and CWD editing.
  </Card>
  <Card title="In-TUI session editing" icon="pencil">
    Press `e` on any session row to edit name, cwd, command, kind, or
    env vars without leaving the TUI. CWD editing opens the same
    find/tree browser. All changes are comment-preserving writes to
    your TOML file.
  </Card>
</CardGrid>

## How it works

```
  ┌──────────┐  SSH   ┌──────────┐  SSH   ┌──────────┐
  │  Phone   │───────▶│  Desktop │◀───────│  Laptop  │
  │ (Termux) │        │ (server) │        │          │
  └──────────┘        └─────┬────┘        └──────────┘
                            │
                     ┌──────┴──────┐
                     │ tmux/zellij │  sessions persist
                     │   sessions  │  across all devices
                     ├─────────────┤
                     │ ● claude    │  ← pa claim (takeover)
                     │ ● dev       │  ← pa launch dev
                     │ ○ tests     │  ← not started yet
                     └─────────────┘
```

<Aside type="tip">
Sessions live in the multiplexer, not in `pa`. When you close your
SSH connection, they keep running. `pa claim` from any device
re-attaches and reshapes the terminal to the new screen size.
</Aside>

## See it in action

<Tabs>
  <TabItem label="Phone bootstrap">
    ```sh
    # SSH into your desktop from your phone (Termux)
    ssh desktop
    cd ~/code/new-project

    # Scaffold a workspace — no editor required
    pa init
    pa add claude -c "claude --resume" --kind claude-code
    pa add dev    -c "bun run dev"     --kind dev-server

    # Launch the TUI, pick a session, Enter
    pa
    ```
  </TabItem>
  <TabItem label="Cross-device takeover">
    ```sh
    # On your phone — start a Claude Code session
    pa launch claude

    # Later, on your laptop — take it over
    pa claim claude
    # Session reshapes to laptop's screen size instantly

    # Even later, back on phone
    pa claim claude
    # Back to phone-sized terminal, context intact
    ```
  </TabItem>
  <TabItem label="TUI workflow">
    ```sh
    # Open the TUI from anywhere
    pa

    # Workspace picker (home screen)
    #   ↑/↓  navigate    Enter  open workspace
    #   n    find folder + scaffold new workspace
    #   ?    help

    # Session list
    #   Enter  attach/create session
    #   e      edit session fields
    #   d      delete session     x  kill live session
    #   Esc    back to picker     q  quit
    ```
  </TabItem>
</Tabs>

## Architecture

<Aside type="note" title="Three-tier config">
portagenty merges configuration from three layers. Any tier can
register projects and sessions; the workspace layer is the primary
authoring surface.
</Aside>

```
  Priority        Tier                      Location
  ─────────────────────────────────────────────────────────────
  highest    ┌─────────────┐     *.portagenty.toml (committable)
             │  Workspace  │     sessions, projects, mpx choice
             └──────┬──────┘
                    │ merges with
             ┌──────┴──────┐     portagenty.toml at project root
  medium     │ Per-project │     sessions defined by the repo
             └──────┬──────┘
                    │ merges with
             ┌──────┴──────┐     $XDG_CONFIG_HOME/portagenty/
  lowest     │   Global    │     default mpx, workspace registry
             └─────────────┘
```

The workspace file is designed to commit — no absolute paths, no
machine-specific state, no secrets. Volatile state (recency, last-
attached timestamps) lives separately in `$XDG_STATE_HOME`.

## Status

**v1 is feature-complete and usable daily.** Core flows work
end-to-end: walk up to find a workspace, render the session list,
pick one, attach — in tmux or zellij.

Most v1.x items have shipped: zellij adapter, untracked adoption,
`pa claim` cross-device takeover, `kind:` hints, per-session env
vars, declarative export, first-run wizard, bundled snippets,
full no-editor session management, workspace picker home screen,
in-TUI find-folder + scaffold with nucleo fuzzy ranking, tree
browser, workspace-scoped session names, in-TUI session editing,
CWD browse via find overlay, responsive 2-line footer, arrow keys
+ Alt+J/K navigation, help overlay, and auto-re-register on
walk-up. Tags + custom-groups views and `pa up` eager-launch are
still on the [roadmap](https://github.com/cybersader/portagenty/blob/main/ROADMAP.md).

WezTerm is **intentionally not supported** — its mux is built around
the GUI emulator's window model, not the headless detach/reattach
pattern portagenty's cross-device work depends on. See the roadmap
for the rationale.

The [design doc](https://github.com/cybersader/portagenty/blob/main/DESIGN.md)
is the source of truth for terminology and architecture.
